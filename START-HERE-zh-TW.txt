==================================================================
  TMRP 家教媒合與評價平台 —— 一鍵試用說明
==================================================================

這份說明教您在一台全新的 Windows 電腦上，點兩下就把整個系統跑起來，
玩完再點兩下完全還原。過程中會「自動重新開機一次」，這是正常的，請不用擔心。


------------------------------------------------------------------
 開始之前，請確認
------------------------------------------------------------------
  1. 一台 Windows 10 或 Windows 11 電腦。
  2. 電腦可以連上網路（過程需要下載軟體與相依套件，約 2 GB）。
     建議使用一般家用網路；若使用公司／學校會封鎖外部網站的受限網路，
     下載可能會失敗（詳見最後「萬一遇到狀況」一節）。
  3. 請預留約 15 GB 的可用磁碟空間（安裝 Docker、下載映像與建置都會用到）。
  4. 請預留 20 ~ 30 分鐘，並讓電腦保持接電、不要關機。
  5. ★ 過程中會「自動重新開機一次」，請先存檔並關閉其他正在使用的程式，
     以免未存檔的工作遺失。★
  6. 建議將本資料夾放在「全英文、不含空格」的路徑下（例如 C:\TMRP），
     以避免中文使用者名稱或特殊符號在少數環境造成 Docker 掛載問題。


------------------------------------------------------------------
 啟動系統（共五步）
------------------------------------------------------------------
  步驟 1. 在本資料夾中，找到「setup.bat」，對它點兩下（滑鼠左鍵雙擊）。
          ※ 若彈出「Windows 已保護您的電腦」或「是否要執行這個檔案」的視窗，
            請點「其他資訊」→「仍要執行」（或按「執行」）。檔案來自網路下載時
            出現此提醒屬正常現象。

  步驟 2. 螢幕會跳出一個藍色視窗問「是否允許變更」（使用者帳戶控制 / UAC），
          請按「是」。
          ※ 若您登入的是「一般使用者帳號」（常見於公司／學校電腦），這個視窗
            會改為要求「輸入管理員帳號與密碼」。請輸入後按確定即可；若您沒有
            管理員密碼，請洽資訊人員協助。

  步驟 3. 程式會自動安裝必要元件，然後「自動重新開機一次」。
          ★ 這是正常的，請放心。★
          重開機後，請像平常一樣登入 Windows，程式多半會自動繼續，
          不需要您再點任何東西。
          ※ 若您登入的是「一般使用者帳號」（常見於公司／學校的電腦），
            重開機後可能會「再跳一次」是否允許變更（UAC）的視窗，請按「是」
            並依指示輸入管理員帳密。這也是正常的，按下去就會繼續。
            （萬一不小心按了「否」也不用擔心，程式不會一直跳窗；想繼續時，
             重新對 setup.bat 點兩下即可。）

  步驟 4. 接著會自動下載並啟動系統，期間黑色視窗會一直顯示進度。
          第一次大約需要 10 ~ 20 分鐘，請耐心等待，不要關掉視窗。

  步驟 5. 完成後，瀏覽器會「自動打開」系統首頁：

                    http://localhost:41080

          同時黑色視窗會顯示一組「管理員帳號與密碼」，
          ★ 請把它抄下來或拍照，這是登入管理員後台用的。★


------------------------------------------------------------------
 如何登入試用
------------------------------------------------------------------
  ● 管理員：使用「步驟 5 畫面上顯示」的帳號與密碼。

  ● 也可以直接用以下兩組現成的測試帳號，體驗家教媒合流程：

        老師端：帳號 tutor    密碼 TutorDemo2026
        家長端：帳號 parent   密碼 ParentDemo2026

  ● 請使用自動開啟的網址 http://localhost:41080 登入。
    請勿改用「區域網路 IP」（例如 http://192.168.x.x:41080）從別台裝置連入，
    那樣登入會無法保持（一登入就被登出）。


------------------------------------------------------------------
 試用結束、完全還原電腦
------------------------------------------------------------------
  1. 在本資料夾中，對「teardown.bat」點兩下。
  2. 同樣按「是」允許變更（UAC）。
  3. 畫面會列出將被移除的項目，請輸入大寫的「Y」再按 Enter 確認。
  4. 程式會關閉系統、解除安裝相關軟體，並「再重新開機一次」完成清理。

  完成後，電腦會回到您試用前的乾淨狀態。

  ※ 註：teardown 只移除「由 setup 安裝的東西」。若這台電腦在試用前就已經
    裝有 Docker 或 WSL，或安裝紀錄已遺失，teardown 會「保留」Docker 與 WSL
    不予移除（畫面上會提示），以免誤刪您原本就在使用的軟體。若您確定要一併
    移除，可自行到「設定 > 應用程式」解除安裝 Docker，並執行「wsl --uninstall」。

  ※ 註：清除過程若需要重新移除 Docker，可能會用到網路或 setup 先前下載的
    安裝檔。若這台電腦此時無法連網、且找不到先前的安裝檔，畫面會提示請您
    自行到「設定 > 應用程式」手動解除安裝 Docker。


------------------------------------------------------------------
 萬一遇到狀況
------------------------------------------------------------------
  ● 瀏覽器打開後顯示「無法連線」：
      請再等 1 ~ 2 分鐘，然後按瀏覽器的重新整理（F5）。第一次啟動較慢。

  ● 黑色視窗出現「Docker engine did not become ready」之類訊息：
      多半是這台電腦的 BIOS 沒有開啟「虛擬化（Virtualization / VT-x / AMD-V）」。
      此項目無法由程式自動開啟，需重開機進入 BIOS 設定頁手動開啟後，
      再重新點一次 setup.bat。若不熟悉 BIOS，可請資訊人員協助。

  ● 黑色視窗出現「Could not enable the Windows virtualization platform」：
      多半是這台電腦受公司／學校的群組原則或 MDM 控管，禁止啟用 Windows 功能。
      此時程式「不會」重新開機，而會直接停在錯誤畫面。請改用個人電腦，
      或請資訊人員協助開啟「適用於 Linux 的 Windows 子系統」與「虛擬機器平台」
      兩項功能後，再重新點一次 setup.bat。

  ● 黑色視窗出現「toomanyrequests」或「rate limit」、或下載映像中途失敗：
      多半是這個網路的公用 IP 觸發了 Docker Hub 的匿名下載流量限制
      （常見於學校／公司共用網路）。請改用其他網路（例如手機個人熱點），
      或稍後再試，然後重新點一次 setup.bat 即可續跑。

  ● 黑色視窗在「建置（Building）」階段出現 pip、npm、PyPI、npmjs，或
    「could not resolve host」、下載套件逾時等訊息：
      建置時除了 Docker Hub，系統還需連到 PyPI 與 npm 套件庫下載相依套件。
      多半是這個網路擋住或限流了這些來源（常見於學校／公司網路）。
      請改用其他網路（例如手機個人熱點），再重新點一次 setup.bat 即可續跑。

  ● 黑色視窗出現「no space left on device」或類似磁碟空間不足的訊息：
      表示建置過程把磁碟空間用完了。請清出更多可用空間（整個過程建議至少
      保留 15 GB），再重新點一次 setup.bat 即可續跑。

  ● 黑色視窗出現「port is already allocated」或「address already in use」：
      表示這台電腦已有別的程式占用了系統要用的 41080 連接埠。
      請關閉可能占用該埠的其他程式（或直接重開機一次清掉殘留），
      再重新點一次 setup.bat 即可。

  ● 黑色視窗出現「Pool overlaps」或「address pool」之類的網段訊息：
      多半是這台電腦正連著公司／學校 VPN，或已有其他 Docker 專案，
      占用了系統要用的內部網段（172.28.0.x）。
      請先「中斷 VPN 連線」並關閉其他 Docker 專案，再重新點一次 setup.bat。
      若仍無法解決，請把這段訊息回報給專案團隊協助調整網段。

  ● 可以打開首頁，但一登入就馬上被登出：
      多半是用了「區域網路 IP」連入。請改用本機自動開啟的網址
      http://localhost:41080 登入即可。

  ● 中途不小心關掉視窗、想重跑、或重開機後沒有自動繼續：
      直接再對「setup.bat」點兩下即可，程式會接續處理。
      若重開機後遲遲沒有自動繼續，請先自行手動重新開機一次，
      再對「setup.bat」點兩下即可續跑。
